# 构建现代 AI 量化交易工作框架

很多从事 A 股量化交易的个人开发者，都会面临一个极具代表性的“痛点”：**QMT（迅投/华泰）只能运行在 Windows 上，但日常开发、研究和 AI 协作，我们更倾向于 macOS。**

尤其对于追求极致体验的开发者来说，如何既保留 Mac 优雅的研究环境，又确保 Windows 端实盘交易的稳定性？本文将系统性地为你梳理一套「AI 量化研究工作站架构」。

## 一、 核心架构：解耦研究与执行

最科学的方案是“双机并行，职能分离”。不要试图在运行实盘交易的 Windows 机器上做繁重的回测或 AI 训练，那会严重威胁实盘的稳定性。

### 1. Mac 主力机：研究与开发中心

- **工具：** VS Code / Cursor / PyCharm, JupyterLab
- **引擎：** `vectorbt`, `Polars`, `DuckDB`
- **大脑：** AI Agent (Claude / GPT), LLM 因子挖掘
- **职责：** 数据研究、策略编写、大规模回测、因子挖掘。

### 2. Windows 挂机：交易执行节点

- **工具：** 华泰 QMT (极简模式 / xtquant)
- **职责：** 接收指令、实盘下单、风控检查、订单状态维护。

> **核心思想：** Mac 负责“想”，Windows 负责“做”。两者通过 HTTP API、Redis 或轻量级 RPC 进行通信。

------

## 二、 为什么 Mac 是量化研究的首选？

虽然 Windows 是交易的“刚需”，但 Mac 在研究端有着无可比拟的优势：

- **AI 工具生态：** Cursor、Claude Code 等 AI 编程助手在 macOS 上的响应速度与集成度极高。
- **Unix 原生环境：** Python 环境管理、Docker、Shell 脚本等工具链极其丝滑。
- **硬件性能：** Apple Silicon 的统一内存架构在处理大规模 DataFrame 时，吞吐能力惊人，且功耗极低，适合长时间运行回测。

------

## 三、 硬件方案建议

推荐“Mac 主力 + Mini PC 挂机”组合：

1. **Mac 端：** 建议 M1/M2/M3 系列，内存 16GB 起步（若做大数据研究，32GB 或更高更佳）。
2. **Windows 端：** 无需高配，一台 Intel NUC 或国产 Mini PC（如零刻、铭凡），i5 + 16G 内存足以稳定跑 QMT。

------

## 四、 框架之争：vectorbt vs backtrader

在回测框架的选择上，开发者常在两类思维中徘徊：

### 1. Backtrader：模拟真实交易的“模拟器”

- **原理：** 事件驱动（Event Driven）。逐根 K 线模拟，适合复杂的订单逻辑、部分成交、滑点模拟。
- **缺点：** 速度慢。

### 2. vectorbt：高速量化研究的“矩阵引擎”

- **原理：** 向量化计算（Vectorized）。利用 NumPy 和 Pandas 进行矩阵运算。
- **优势：** **极快**。它将策略信号视为一连串的 $0$ 和 $1$，一次性计算全样本收益。

**为什么 AI 时代更偏向 vectorbt？**

AI 的本质是数学和矩阵。LLM 生成 SQL 风格或矩阵运算逻辑的代码远比编写复杂的事件循环类代码更准确。对于海量参数搜索和因子挖掘，vectorbt 是唯一的选择。

------

## 五、 现代 AI 量化技术栈：数据基石

现代量化系统正在经历从“Pandas 时代”向“高性能列式时代”的迁移。

### 1. 核心四剑客

- **Polars：** 下代 DataFrame 引擎，Rust 编写，比 Pandas 快 5-50 倍。
- **DuckDB：** 嵌入式分析数据库，堪称“分析版 SQLite”，处理 Parquet 文件极快。
- **Arrow：** 统一的内存标准，实现不同工具间的“零拷贝”数据传递。
- **Parquet：** 高压缩比的列式存储格式，量化历史数据的标配。

| **工具**   | **角色**   | **优势**                      |
| ---------- | ---------- | ----------------------------- |
| **Polars** | 数据清洗   | 多线程并行，内存效率极高      |
| **DuckDB** | 结构化查询 | 适合 SQL 风格的大规模数据过滤 |
| **Arrow**  | 数据中转   | 消除数据在内存中搬运的损耗    |

------

## 六、 未来趋势：AI Agent 驱动的研究系统

未来的量化不再是“人工写策略”，而是构建一套“自动研究系统”：

1. **LLM Agent** 根据市场逻辑生成 1000 个原始因子。
2. **Polars/DuckDB** 在秒级内完成全市场因子的数据计算。
3. **vectorbt** 进行海量参数回测与筛选。
4. **生成报告** 并自动推送指令给 Windows 端的 **QMT** 执行。

------

## 七、 总结

对于个人开发者，最合理的配置是：**Mac 做研究端（大脑），Windows 做执行端（手脚）。**

通过 **Parquet → Arrow → Polars → vectorbt → LLM Agent → QMT** 这一链路，你构建的不仅是一个交易系统，而是一个具备现代数据处理能力的“量化实验室”。

> **记住：** 真正重要的不是学习某一个框架的 API，而是建立**向量化思维**与**研究/执行分离**的架构意识。
